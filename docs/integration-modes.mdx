---
title: "Integration modes"
description: "Proxy Hopper supports three ways for clients to send requests. All three use the same IP rotation and retry logic."
---

## Overview

| Mode | How to use | Retry HTTPS? |
|---|---|---|
| **HTTP proxy** | Set `http_proxy=http://proxy-hopper:8080` | Yes |
| **CONNECT tunnel** | Set `https_proxy=http://proxy-hopper:8080` | No |
| **URL forwarding** | Use `X-Proxy-Hopper-Target` header | Yes |

## HTTP proxy mode

Standard HTTP proxy protocol. Proxy Hopper receives the full request, forwards it through an external proxy IP, and streams the response back.

Works with any HTTP client that supports proxy settings.

<CodeGroup>

```bash curl
curl --proxy http://localhost:8080 http://example.com
```

```python requests
import requests
session = requests.Session()
session.proxies = {
    "http": "http://localhost:8080",
}
resp = session.get("http://example.com")
```

```python aiohttp
import aiohttp
async with aiohttp.ClientSession() as session:
    async with session.get(
        "http://example.com",
        proxy="http://localhost:8080"
    ) as resp:
        print(resp.status)
```

</CodeGroup>

## CONNECT tunnel mode

For HTTPS, HTTP clients issue a `CONNECT` request to establish a tunnel. Proxy Hopper picks an external proxy IP, establishes the tunnel through it, and relays bytes in both directions.

<Warning>
  CONNECT tunnels are opaque — once the TLS handshake completes, Proxy Hopper is relaying encrypted bytes it cannot inspect. **It cannot retry a mid-flight failure.** If the connection drops after the tunnel is established, the error is returned directly to the client.

  Use forwarding mode if you need retry support over HTTPS.
</Warning>

<CodeGroup>

```bash curl
curl --proxy http://localhost:8080 https://example.com
```

```python requests
import requests
session = requests.Session()
session.proxies = {
    "https": "http://localhost:8080",
}
resp = session.get("https://example.com")
```

</CodeGroup>

## URL forwarding mode

The recommended mode for HTTPS APIs. Set the `X-Proxy-Hopper-Target` header to the target scheme and host, then send requests to Proxy Hopper as if it were the target server. Proxy Hopper reconstructs the full URL and makes the HTTPS request itself — owning the connection end-to-end.

This enables full retry support: on a 429 or 5xx, Proxy Hopper can pick a different IP and retry without the client knowing.

<CodeGroup>

```bash curl
curl -H "X-Proxy-Hopper-Target: https://api.example.com" \
     http://localhost:8080/v1/endpoint?q=search
```

```python requests
import requests
session = requests.Session()
session.headers["X-Proxy-Hopper-Target"] = "https://api.example.com"

# Use normal paths — the header is never affected by URL manipulation
resp = session.get("http://localhost:8080/v1/endpoint", params={"q": "search"})
# → forwards to https://api.example.com/v1/endpoint?q=search
```

```python aiohttp
import aiohttp
async with aiohttp.ClientSession(
    headers={"X-Proxy-Hopper-Target": "https://api.example.com"}
) as session:
    async with session.get("http://localhost:8080/v1/endpoint") as resp:
        print(resp.status)
```

</CodeGroup>

The header value may include a base path (`https://api.example.com/v2`) which is prepended to the request path.

### Disabling modes

By default all three modes are enabled. Restrict to specific modes via config or env var:

```yaml
server:
  modes:
    - forwarding
    - connect_tunnel
```

```bash
PROXY_HOPPER_MODES=forwarding,connect_tunnel
```
